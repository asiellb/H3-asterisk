Project


The Project Definition form is used to edit or change project information.

When changing project information, it is recommended that an Active project be blocked from processing.


Field Definitions

Description

A Unique name of the project. Up to 20 alphanumaric characters including a-z, 0-9, dash("-") and underscore("_") characters can be used.

Status

Shows the current state of the project.

  • Active - Project will be processed at the specified date and time.
  • Blocked - Project is currently suspended or blocked from processing.

Testcall Approved

Displays the date and time of the last testcall approved. At least one testcall must be approved before the project can be set to Active.

Start/Stop Date

Defines the dates on which the project must start and stop. Projects can be setup to span multiple days, weeks or even months. If a project spans more than one day, dialing will start and stop each day using the Callcenter time start/stop and the Local Customer Time start/stop times.

Max Lines to Use

Defines the maximum number of lines to be used when dialing lead phone numbers for the project. The time it takes to complete a project is a product of the length of the message in seconds, the number of leads to dial and the number of lines used.

Using more lines will result in the project completing sooner. For example, with a message length of 45 seconds, we recommend estimating 64 dials per hour per line. With 100,000 leads to be dialed over a 5 hour period (i.e. 9am-2pm), use this equation to calculate the number of lines required:

Maxlines = 100,000 / 5 hrs / 64 calls per hour = 313 lines required

RECOMMENDATION: Check the number of completed dials after the first hour, and increase the number of Maxlines to speed up the project, or decrease the value to slow the project down.

Max Calls Per Day

Defines the maximum number of calls to be made per day for this project. Applicable only to projects that span more than one day. Dialing will continue through the day until the number of calls in this field have been completed. When this limit is reached, the  dialing will stop for the day, the project will remain active, and dialing will continue on the next scheduled day.

Workday Start/Stop Time

The times (expressed in your time zone) that define your workday. This is particularly important for Press-1 type projects and should correspond to the call center's operating hours. If your project is not a press-1 type, then some suitable wide period (e.g. 07:00 - 23:00) should be used.

Prospect timezone start/stop

These are the times (expressed in the timezone of the party being called) that define the period that the prospects will be called (in the prospects local timezone as determined from the area code).
For example: if you wish to call prospects between 4:00pm and 8:00pm and you are in the eastern time zone, then you should define your workday as 16:00-23:00 and prospects start 16:00 and stop at 20:00. Note that the workday ends at 23:00 Eastern since that is 8pm pacific, because you might be calling pacific-based prospects.
It is rarely necessary (and sometimes illegal) to call prospects after 21:00

Delivery type

Selects the type of dial project to be performed. Use the radio buttons to select one of the options:

  • Voice Delivery plays a pre-recorded message
  • Press 1 Allows Live answered prospects to press the “1” key on their phone to be forwarded to the project Callcenter and placed in contact with a live agent. Pressing the “2” key places the phone number in a local customer Do-Not-Call list.
  • Survey permits the live answered calls to answer a series of questions using their keypad. Each answer is a single digit keypress.
  • Cold Calling every connect is immediately bridged to the call center

Message Type

Selects the tpe of call process handling to be performed based on how the call is answered:

  • Voicemail Voicemail message delivery for all machine answered calls. Live answered calls are detected and dropped.
  • Live Message delivery to only live answered calls. Machine answered calls are detected and dropped.
  • Both The appropriate message is played to the answered call based on a detected live or machine answered status.

Phone Number of Callcenter

10-digit phone number of the Callcenter to be used by dialers when processing "Press 1" projects. This is the phone number used by the dialers to connect interested lead contacts that have pressed "1" on their telephone keypads.

Press 1 Senario: When the project starts dialing, each answered call will have the appropriate pre-recorded message played. The message should contain a phrase similar to "press 1 to be connected to one of our agents now". Upon pressing the "1", the dialer will detect the tone, and automatically dial the "Phone Number of Callcenter" where an available Customer Agent will pick up the phone and be connected to the interested lead. This process is repeated until all agents are busy on the phone with responding leads. When all Agents are busy, the dialers will suspend dialing. When an Agent hangs up, or finishes with a lead, the dialers will sense this termination of the call and once again begin dialing in search of another interested lead.

Number of Customer Agents

Defines to the dialers the number of Customer Agents, at the Callcenter, available to take "Press 1" responding lead contacts.

Lines per Agent

Defines to the dialers the number of telephone lines that each Customer Agent can handle simultaneously. The value entered should be one plus the number of lines that can be placed on hold.

Weekend

Defines to the dialers how a project is to be processed over the weekend days, Saturday and Sunday.

  • No - No dialing to take place over the weekend. For projects that span multiple weeks and only on M-F.
  • Only Saturday - Dialing will take place on Saturday only. Use this option for Saturday Only projects or projects that are to be dialed Monday through Saturday.
  • Only Sunday - Dialing will take place on Sunday.
  • Both - Dialing will occur on both Saturday and Sunday.

User

Shows the assigned user who created the project.
